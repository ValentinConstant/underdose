3.1 Le modèle utilisé : la forêt aléatoire (CART)

Forêt de 50 arbres. Ici la question n’est pas d’optimiser la forêt mais de comprendre son fonctionnement interne.

3.1.1 Division des données en 2 : données d’entrainements (75%) et données de test (25%)

3.1.1.1 Pour le dataset n°1 (4*CMI)

## 75% of the sample size
smp_size <- floor(0.75 * nrow(df))

## set the seed to make your partition reproducible - similar to random state in Python
set.seed(123)
train_ind <- sample(seq_len(nrow(df)), size = smp_size)

train_df <- df[train_ind, ]
test_df <- df[-train_ind, ]

3.1.1.2 Pour le dataset n°1 (1*CMI)

## 75% of the sample size
smp_size <- floor(0.75 * nrow(df_1))

## set the seed to make your partition reproducible - similar to random state in Python
set.seed(123)
train_ind <- sample(seq_len(nrow(df_1)), size = smp_size)

train_df_1 <- df_1[train_ind, ]

3.1.2 Création du modèle

Le package “caret” est utilisé pour constuire le modèle.

3.1.2.1 Forêt n°1 (4*CMI)

model_rf <- caret::train(ECOFF_4CMI ~ ., data = train_df,method = "rf", #random forest
trControl = trainControl(method = "repeatedcv", number = 10,repeats = 5, verboseIter = FALSE))

3.1.2.2 Forêt n°2 (1*CMI)

model_rf_1 <- caret::train(ECOFF_1CMI ~ ., data = train_df_1,method = "rf", #random forest
trControl = trainControl(method = "repeatedcv", number = 10,repeats = 5, verboseIter = FALSE))
